PURPOSE
--------------------------------------------------------------------------------
This block creates AndroidManifest.xml. It declares all permissions the app
needs, registers every Activity, Service, BroadcastReceiver, and Provider, and
sets the application-level attributes. After this block, the APK can be
installed and the OS knows the full surface area of the app. No Kotlin classes
are created yet — the manifest references class names that will be written in
later blocks.

--------------------------------------------------------------------------------
FILES TO CREATE / MODIFY
--------------------------------------------------------------------------------

FILE: apk/app/src/main/AndroidManifest.xml
Role: Android OS contract — permissions, components, intent filters.

  Manifest attributes:
    package="com.baxter.schedulaizer"
    xmlns:android="http://schemas.android.com/apk/res/android"

  PERMISSIONS — each listed with its justification:

    android.permission.INTERNET
      Justification: BudgetBlasterBridge POSTs to homelab LAN HTTP endpoint.
                     FileTransferManager uses RemoteDexter tunnel (TCP).

    android.permission.FOREGROUND_SERVICE
      Justification: SchedulaizerService runs as a foreground service.

    android.permission.FOREGROUND_SERVICE_DATA_SYNC
      Justification: foregroundServiceType="dataSync" on SchedulaizerService.

    android.permission.WAKE_LOCK
      Justification: SchedulaizerService holds a partial wake lock during
                     active file transfers.

    android.permission.RECEIVE_BOOT_COMPLETED
      Justification: BootReceiver reschedules all alerts after device reboot.

    android.permission.CAMERA
      Justification: CaptureFragment uses CameraX to photograph receipts.

    android.permission.RECORD_AUDIO
      Justification: VoiceListenerService uses SpeechRecognizer (mic input).

    android.permission.USE_EXACT_ALARM  (API 33+)
      Justification: AlertScheduler sets exact alarms for time-critical events.
      Note: On API 32 and below, setExactAndAllowWhileIdle() is used instead.

    android.permission.SCHEDULE_EXACT_ALARM
      Justification: Fallback for exact alarm scheduling on API 31-32.

    android.permission.POST_NOTIFICATIONS  (API 33+)
      Justification: Displaying alert notifications requires explicit permission
                     on Android 13+.

    android.permission.ACCESS_NETWORK_STATE
      Justification: FileTransferManager checks network availability before
                     attempting uploads.

--------------------------------------------------------------------------------
SAFETY & SECURITY NOTES
--------------------------------------------------------------------------------
- android:exported="false" on every component except MainActivity. Only the
  launcher activity is intentionally exported.
- FileProvider is used for all file URI sharing — direct file:// URIs are
  prohibited on API 24+ and would cause FileUriExposedException.
- USE_EXACT_ALARM on API 33+ requires user to grant the permission in Settings.
  The app must check AlarmManager.canScheduleExactAlarms() before using exact
  alarms and fall back to inexact WorkManager tasks if permission is denied.
- android:allowBackup="true" is acceptable for personal single-user app. If
  the user ever adds sensitive fields, this should be reviewed.
