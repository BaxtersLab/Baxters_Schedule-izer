PURPOSE
--------------------------------------------------------------------------------
This block implements the Bills tab: BillsFragment (bill list with overdue
highlighting and monthly total), BillsViewModel (data and state), and
BillEditActivity (create/edit form). After this block, the user can manage
all recurring bills, mark them paid (which auto-advances the cycle), and see
a summary of what is owed this month.

FILE: app/src/main/java/com/baxter/schedulaizer/ui/bills/BillsViewModel.kt
Role: State for the Bills tab.

  Class: BillsViewModel(application: Application) : AndroidViewModel(application)

  private val billRepo = SchedulaizerApp.get(application).billRepository

  State flows:
    private val _showPaidBills = MutableStateFlow(false)

    val bills: StateFlow<List<BillEntity>>
      Combines billRepo.allBills / billRepo.unpaidBills based on _showPaidBills.
      When _showPaidBills = false: uses billRepo.unpaidBills.
      When _showPaidBills = true: uses billRepo.allBills.
      stateIn(viewModelScope, SharingStarted.Lazily, emptyList())

    val totalDueThisMonthCents: StateFlow<Long>
      = flow { emit(billRepo.totalUnpaidCentsUpTo(DateUtils.endOfMonthMs())) }
        .stateIn(viewModelScope, SharingStarted.Lazily, 0L)
      Re-emits when bills change by combining with bills flow.

  Functions:
    fun setShowPaidBills(show: Boolean) — updates _showPaidBills
    suspend fun markPaid(billId: Long)  — calls billRepo.markPaid(billId)
    suspend fun deleteBill(bill: BillEntity) — calls billRepo.delete(bill)


FILE: app/src/main/java/com/baxter/schedulaizer/ui/bills/BillsFragment.kt
Role: Displays the bill list with overdue/due-soon highlighting and totals banner.

  Class: BillsFragment : Fragment(R.layout.fragment_bills)

  UI components (FragmentBillsBinding):
    recyclerViewBills: RecyclerView
    textTotalDue: TextView            — "Due this month: $X.XX"
    switchShowPaid: SwitchMaterial    — toggle to show/hide paid bills
    btnAddBill: FloatingActionButton  — open BillEditActivity (new bill)

  onViewCreated():
    Binds ViewModel.
    RecyclerView with BillAdapter (vertical list).
    Observes viewModel.bills -> submits to BillAdapter.
    Observes viewModel.totalDueThisMonthCents -> updates textTotalDue.
    switchShowPaid listener -> viewModel.setShowPaidBills().
    btnAddBill -> navigate to billEditActivity with bill_id = 0L.

  BillAdapter (inner class or BillAdapter.kt):
    Each row shows item_bill.xml.
    Overdue rows: background = R.color.overdueRed.
    Due soon (within 3 days): priority stripe = HIGH color.
    Shows: name, amount, due date, days until due badge, paid checkmark.
    On item click: navigate to BillEditActivity with bill_id = bill.id.
    On "Mark Paid" button tap: viewModel.markPaid(bill.id).
    On item long press: confirmation dialog -> viewModel.deleteBill(bill).

--------------------------------------------------------------------------------
DATA FLOW
--------------------------------------------------------------------------------
User taps "Mark Paid" in BillAdapter:
  BillsFragment -> viewModel.markPaid(id)
  -> viewModelScope.launch(IO) { billRepo.markPaid(id) }
  -> BillRepository.markPaid() [marks paid, advances nextDueMs]
  -> Room emits updated bills list
  -> BillsViewModel.bills StateFlow re-emits
  -> BillAdapter diff updates the row (paid checkmark, new due date)

Monthly total banner:
  bills StateFlow changes -> totalDueThisMonthCents re-evaluated
  -> textTotalDue updated with new total

--------------------------------------------------------------------------------
SAFETY & SECURITY NOTES
--------------------------------------------------------------------------------
- editNotes field: placeholder text explicitly says "(bank name, etc.)".
  Do NOT hint or suggest storing passwords, card numbers, or PINs.
- Amount parsing from text input: always use BigDecimal for parsing the display
  string before converting to cents:
    val cents = (BigDecimal(editAmount.text.toString()).setScale(2)
                 .multiply(BigDecimal(100))).toLong()
  This avoids floating-point precision errors when converting "$45.99" -> 4599.
