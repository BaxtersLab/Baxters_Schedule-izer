PURPOSE
--------------------------------------------------------------------------------
This block creates the enums, constants, and the Application subclass that
every other part of the app depends on. After this block, all shared vocabulary
(priority levels, categories, recurrence types, notification channel IDs,
intent action keys) is defined in one place. The Application class sets up
the Room database singleton so ViewModels and Services can access it.

This block does NOT define data classes or database schema (Module D), utility
functions (C-2), or any UI components.

FILE: app/src/main/java/com/baxter/schedulaizer/util/PriorityLevel.kt
Role: Defines the four-tier priority system used on every event, bill, and memo.

  Enum class: PriorityLevel
    Values (in descending urgency order):
      CRITICAL  — value 0 — color hint: red (#D32F2F)
                  Use: overdue bills, same-day critical appointments
      HIGH      — value 1 — color hint: orange (#F57C00)
                  Use: upcoming bills within 3 days, important appointments
      NORMAL    — value 2 — color hint: blue (#1565C0)
                  Use: standard calendar events and bills
      LOW       — value 3 — color hint: grey (#757575)
                  Use: optional reminders, nice-to-have tasks

    Companion object:
      fun fromInt(value: Int): PriorityLevel
        Maps an integer (stored in Room) back to the enum constant.
        Returns NORMAL if the value is out of range — never throws.

    Property: colorRes — a suggested Android color resource hint stored as
              a hex string. Not an actual @ColorRes — just documentation.
    Property: displayName — human-readable string for UI labels.

FILE: app/src/main/java/com/baxter/schedulaizer/SchedulaizerApp.kt
Role: Application subclass. Creates the Room database singleton and provides
      global access to repositories.

  Class: SchedulaizerApp : Application()

    Properties (lazy):
      database: SchedulaizerDatabase
        Created via Room.databaseBuilder(this, SchedulaizerDatabase::class.java,
        "schedualizer.db").build()
        Lazy — not created until first access.

      eventRepository: EventRepository
        Instantiated with database.eventDao()

      billRepository: BillRepository
        Instantiated with database.billDao(), database.alertDao()

      attachmentRepository: AttachmentRepository
        Instantiated with database.attachmentDao()

    companion object:
      fun get(context: Context): SchedulaizerApp
        Returns (context.applicationContext as SchedulaizerApp)
        Used by ViewModels and Services to access repositories without
        passing context chains.

    onCreate():
      Call super.onCreate().
      Create all notification channels by calling a private
      createNotificationChannels() method.

    createNotificationChannels() — private:
      Creates three NotificationChannels (API 26+):
        1. CHANNEL_ALERTS: importance IMPORTANCE_HIGH, name "Reminders & Alerts"
        2. CHANNEL_SERVICE: importance IMPORTANCE_LOW, name "Schedualizer Service"
        3. CHANNEL_TRANSFER: importance IMPORTANCE_DEFAULT, name "File Transfers"
      Registers them via NotificationManager.createNotificationChannels().
      Safe to call multiple times — existing channels are not modified.

INTERFACES EXPOSED

  are needed (ViewModels, Services, Receivers).

--------------------------------------------------------------------------------
DESIGN DECISIONS
--------------------------------------------------------------------------------
- Priority stored as Int in Room, not as a String. Allows ORDER BY priority ASC
  to sort CRITICAL first without string comparison.
- Notification channels created in Application.onCreate(), not in Service or
  Activity, to ensure they exist before any notification is posted.
- SchedulaizerApp uses lazy {} for database and repositories — avoids creating
  Room instances at static init time, which can cause ANR on some devices.

--------------------------------------------------------------------------------
SAFETY & SECURITY NOTES
--------------------------------------------------------------------------------
- AppConstants.PREF_BUDGET_BLASTER_ENDPOINT stores a LAN IP. This value is
  never logged or included in crash reports.
- No hardcoded IP addresses in AppConstants — all defaults are obviously local
  (192.168.x.x) and clearly labeled as defaults to be changed by the user.
