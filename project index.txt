================================================================================
WHAT THIS APP IS
================================================================================

A personal Android productivity hub for Baxter. All life data — calendar
events, recurring bills, receipts, voice memos, and reminders — flows through
this app and up to the homelab PC via the RemoteDexter tunnel. The Windows side
is handled by Budget Blaster (OCR + accounting). Schedualizer is the mobile
input surface.

================================================================================
INTEGRATION POINTS
================================================================================

RemoteDexter (stand-alone backup)
  - Schedualizer reuses the compiled Rust .so from RemoteDexter-v3.
  - The .so provides the encrypted Android-to-Windows tunnel.
  - Schedualizer does NOT modify RemoteDexter source. Read/use only.
  - .so ABI folders needed: arm64-v8a/, armeabi-v7a/, x86_64/

Budget Blaster (Windows app)
  - Schedualizer POSTs receipt images as JSON to a LAN HTTP endpoint.
  - Endpoint is user-configured in Settings (IP:port, no hardcoding).
  - Payload: { date, amount_hint, category, filename, image_base64 }

================================================================================
KEY DESIGN DECISIONS (SUMMARY)
================================================================================

1. Transport:   Reuse RemoteDexter .so — do not re-implement the tunnel.
2. Database:    Room (SQLite), offline-first. No cloud DB.
3. Voice:       Android SpeechRecognizer, EXTRA_PREFER_OFFLINE=true. No audio
                leaves the device.
4. DI:          Manual (no Hilt in v1). SchedulaizerApp provides all repos.
5. BB bridge:   HTTP POST to LAN. HttpURLConnection — no Retrofit/OkHttp.
6. Currency:    Stored as Long cents. No Float/Double for money.
7. ProGuard:    Off in debug. On in release (basic rules).
8. Urgency:     distinctUntilChanged() gates notification rebuilds — no spam.
