PURPOSE
--------------------------------------------------------------------------------
This block implements the Calendar tab: CalendarFragment (month/event list view),
CalendarViewModel (data and filtering), and EventEditActivity (create/edit form).
After this block, the user can view all upcoming events, filter by category and
priority, tap an event to edit it, create new events from a form, and delete events.
Alert scheduling is triggered automatically on save.

FILE: app/src/main/java/com/baxter/schedulaizer/ui/calendar/CalendarViewModel.kt
Role: State holder for the Calendar tab. Exposes filtered event lists and
      handles user actions.

  Class: CalendarViewModel(application: Application) : AndroidViewModel(application)

  Access to repositories via SchedulaizerApp.get(application):
    private val eventRepo = SchedulaizerApp.get(application).eventRepository

  State flows:
    private val _filterCategory = MutableStateFlow<EventCategory?>(null)
    private val _filterPriority = MutableStateFlow<Int?>(null)
    private val _selectedMonth = MutableStateFlow(DateUtils.startOfMonthMs())

    val events: StateFlow<List<EventEntity>>
      Combines eventRepo.allEvents with _filterCategory and _filterPriority.
      Uses combine() from kotlinx.coroutines.flow.
      Filter logic:
        - If _filterCategory != null: only include events matching category.
        - If _filterPriority != null: only include events with priority <= value.
      Stateflow produced by stateIn(viewModelScope, SharingStarted.Lazily, emptyList()).

    val eventsThisMonth: StateFlow<List<EventEntity>>
      Uses eventRepo.getEventsInRange(_selectedMonth, endOfMonth).
      Updated when _selectedMonth changes.

  Functions:
    fun setFilterCategory(cat: EventCategory?) — updates _filterCategory
    fun setFilterPriority(level: PriorityLevel?) — updates _filterPriority
      (passes level?.ordinal)
    fun setSelectedMonth(startMs: Long) — updates _selectedMonth
    suspend fun deleteEvent(event: EventEntity) — calls eventRepo.delete(event)
    fun navigateToEditEvent(fragment: Fragment, eventId: Long = 0L)
      Calls findNavController().navigate() with action and event_id argument.


FILE: app/src/main/java/com/baxter/schedulaizer/ui/calendar/CalendarFragment.kt
Role: Displays event list + month header. Observes CalendarViewModel.

  Class: CalendarFragment : Fragment(R.layout.fragment_calendar)

  UI components (via ViewBinding — FragmentCalendarBinding):
    recyclerViewEvents: RecyclerView   — event list
    textMonthHeader: TextView          — e.g. "June 2026"
    btnPrevMonth: ImageButton          — navigate to previous month
    btnNextMonth: ImageButton          — navigate to next month
    btnAddEvent: FloatingActionButton  — navigate to EventEditActivity (new)
    spinnerCategory: Spinner           — filter by category
    spinnerPriority: Spinner           — filter by priority

  onViewCreated():
    Binds ViewModel. Sets up RecyclerView with EventAdapter.
    Observes viewModel.events and submits to adapter.
    Sets up FAB click -> navigate to eventEditActivity with event_id = 0L.
    Sets up prev/next month buttons -> viewModel.setSelectedMonth().
    Sets up spinner listeners -> viewModel.setFilterCategory() / setFilterPriority().

  RecyclerView adapter (inner class or separate file — EventAdapter):
    Displays each EventEntity in item_event.xml.
    Shows: title, formatted start date/time, category chip, priority color stripe.
    On item click: navigate to EventEditActivity with event_id = event.id.
    On item long press: show confirmation dialog -> viewModel.deleteEvent(event).


FILE: app/src/main/java/com/baxter/schedulaizer/ui/calendar/EventEditActivity.kt
Role: Full-screen form for creating and editing events.

  Class: EventEditActivity : AppCompatActivity()

  Navigation args:
    event_id: Long (from intent extras, 0L = new event)

  UI components (ActivityEventEditBinding):
    editTitle: EditText
    editDescription: EditText
    btnPickDate: Button         — shows DatePickerDialog
    btnPickTime: Button         — shows TimePickerDialog
    checkAllDay: CheckBox       — toggles time field visibility
    spinnerCategory: Spinner    — EventCategory values
    spinnerPriority: Spinner    — PriorityLevel values
    spinnerRecurrence: Spinner  — RecurrenceType values
    seekBarAlert: SeekBar       — alert offset: 0, 5, 10, 15, 30, 60, 120, 1440 minutes
    textAlertSummary: TextView  — "Alert: 15 minutes before"
    btnSave: Button
    btnCancel: Button

  Behavior:
    On load (event_id != 0L): loads EventEntity from eventRepo.getById(),
      populates all fields.
    On load (event_id == 0L): pre-fills date to tomorrow, time to 09:00.
    btnSave: validates title non-empty, endMs >= startMs.
      Calls eventRepo.save(event).
      Calls AlertScheduler.scheduleForEvent(context, savedEvent) if alertOffset > 0.
      Calls setResult(RESULT_OK) and finish().
    btnCancel: calls finish() without saving.
    DatePickerDialog: updates selectedDateMs field.
    TimePickerDialog: updates selectedTimeMs field.
    checkAllDay: hides/shows time fields.
