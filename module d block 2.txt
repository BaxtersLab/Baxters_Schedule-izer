--------------------------------------------------------------------------------
PURPOSE
--------------------------------------------------------------------------------
This block creates the five DAO interfaces — one per entity. DAOs are the only
layer that talks directly to SQLite via Room. All read queries return
Flow<List<T>> so ViewModels can observe live database changes. All write
operations are suspend functions for coroutine-safe execution on the IO
dispatcher. After this block, every data access pattern the app needs is
defined as a typed, compile-checked method.

This block does NOT create the Database class (D-3) or Repositories (D-4).

FILE: app/src/main/java/com/baxter/schedulaizer/data/db/dao/EventDao.kt
Role: All database operations for the events table.

  @Dao interface EventDao

  Queries:

    @Query("SELECT * FROM events ORDER BY startMs ASC")
    fun getAllEvents(): Flow<List<EventEntity>>
      Observes all events sorted by start time ascending.
      UI subscribes to this for the full event list.

    @Query("SELECT * FROM events WHERE startMs >= :fromMs AND startMs <= :toMs ORDER BY startMs ASC")
    fun getEventsInRange(fromMs: Long, toMs: Long): Flow<List<EventEntity>>
      Used by CalendarFragment to load events for a displayed month/week.

    @Query("SELECT * FROM events WHERE startMs >= :fromMs ORDER BY startMs ASC")
    fun getUpcomingEvents(fromMs: Long): Flow<List<EventEntity>>
      fromMs is typically DateUtils.todayStartMs(). Returns all future events.

    @Query("SELECT * FROM events WHERE priority <= :maxPriority ORDER BY priority ASC, startMs ASC")
    fun getEventsByPriority(maxPriority: Int): Flow<List<EventEntity>>
      maxPriority = 0 returns only CRITICAL. maxPriority = 3 returns all.

    @Query("SELECT * FROM events WHERE id = :id")
    suspend fun getEventById(id: Long): EventEntity?
      Returns null if not found. Used by EventEditActivity to load for editing.

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertEvent(event: EventEntity): Long
      Returns the new row ID. Used by EventRepository.save().

    @Update
    suspend fun updateEvent(event: EventEntity)

    @Delete
    suspend fun deleteEvent(event: EventEntity)

    @Query("DELETE FROM events WHERE id = :id")
    suspend fun deleteEventById(id: Long)

    @Query("SELECT COUNT(*) FROM events WHERE startMs >= :fromMs AND startMs <= :toMs")
    suspend fun countEventsInRange(fromMs: Long, toMs: Long): Int
      Used by calendar month grid to show event count badge per day.

--------------------------------------------------------------------------------
DATA FLOW
--------------------------------------------------------------------------------
DAO -> Room -> SQLite

All Flow queries are backed by SQLite invalidation tracking. When a write
occurs on a table, Room automatically invalidates active Flow collectors on
that table, causing them to re-emit the latest data.

Read path:
  DAO.getXxx() -> Flow<List<Entity>> -> Repository transforms/filters ->
  ViewModel StateFlow -> Fragment collects -> UI updates

Write path:
  Fragment user action -> ViewModel -> Repository.save() ->
  DAO.insertXxx(entity) -> Room write -> Flow re-emits -> UI refreshes

--------------------------------------------------------------------------------
SAFETY & SECURITY NOTES
--------------------------------------------------------------------------------
- All SQL queries use Room's parameterized binding (":parameter" syntax).
  No string concatenation in query bodies.
- deleteEventById, deleteMemosForParent, deleteAlertsForParent are paired
  operations — Repository must call all three when deleting an event to
  avoid orphaned records.
